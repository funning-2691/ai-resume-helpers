## 1. 背景与目标

### 1.1 背景
求职者普遍存在以下痛点：
- 不知道如何量化工作成果
- 简历描述与岗位要求不匹配
- 市面上缺少免费、便捷的简历优化工具

### 1.2 目标
提供一个AI驱动的简历优化工具，帮助用户：
- 快速获得针对目标岗位的简历优化建议
- 了解当前简历与岗位的差距
- 回溯历史优化版本

## 4. 功能详情

### 4.1 功能：AI简历优化

| 项目 | 说明 |
| :--- | :--- |
| **功能描述** | 用户输入岗位描述和个人简历，调用大模型API返回优化后的简历内容 |
| **输入** | 岗位描述文本、个人简历文本 |
| **处理逻辑** | 将输入拼接为提示词，调用大模型API，设置temperature参数控制优化强度 |
| **输出** | 优化后的简历文本 |
| **异常处理** | API超时/失败时，前端提示“网络繁忙，请稍后重试”，不丢失用户已输入内容 |

**优化强度设计**：

| 强度 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| 保守 | 保留原意，优化措辞和语法 | 简历已较好，只需微调 |
| 中等 | 适当强化成果描述 | 通用场景 |
| 激进 | 大幅改写，突出亮点 | 简历较弱，需要较大提升 |

> 当前状态：三种强度已在代码层面实现，但界面上缺少可视化选择和说明，后续版本需补充。

### 4.2 功能：匹配度评估

| 项目 | 说明 |
| :--- | :--- |
| **功能描述** | 计算岗位描述与个人简历的关键词匹配程度 |
| **当前算法** | 基于关键词的TF-IDF思想，统计岗位要求中的核心词在简历中的出现密度 |
| **输出** | 0-100%的匹配度分数 |

**当前局限性**：
- 无法理解语义（如同义词“开发”与“研发”无法匹配）
- 仅支持关键词统计，不支持分维度分析

**后续优化方向**：
- 引入向量化模型（Embedding）进行语义匹配
- 细化评估维度：技能匹配、经验年限、教育背景、证书等
- 提供分维度得分和改进建议

### 4.3 功能：历史记录

| 项目 | 说明 |
| :--- | :--- |
| **功能描述** | 自动保存每次优化结果，支持查看和回溯 |
| **技术方案** | SQLite本地数据库，写入服务器文件 |
| **交互方式** | 点击“历史记录”按钮，列表展示所有记录，点击单条查看详情 |

**当前局限**：
- 数据写入服务器端SQLite文件，部署到Vercel后可能因无服务器实例重启而丢失
- 不同设备/浏览器无法同步

**后续优化方向**：
- 迁移至云数据库（如Supabase）
- 增加简单的用户标识（如设备ID或账号体系）

## 5. 已尝试但放弃的功能

### PDF上传解析

| 项目 | 说明 |
| :--- | :--- |
| **尝试内容** | 增加PDF文件上传，自动解析文本内容填入简历框 |
| **遇到的困难** | `pdf-parse` 库与Next.js 15版本存在兼容性问题，调试约2小时未解决 |
| **放弃原因** | 核心功能（文本输入优化）已可跑通，PDF上传为锦上添花，不阻塞MVP上线 |
| **后续计划** | 待技术方案成熟后重新评估，或欢迎社区贡献 |

## 7. 非功能需求

| :--- | :--- |
| 性能 | 单次优化请求，95%响应时间在30秒内 |
| 数据隐私 | 用户输入数据仅用于本次优化和历史存储，不用于模型训练或分享给第三方 |
| 可用性 | 核心功能（优化、历史、导出）可用性目标99% |
